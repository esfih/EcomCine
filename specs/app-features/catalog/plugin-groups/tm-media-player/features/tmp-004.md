# tmp-004 — AJAX Navigation List and Store Content Loaders

## Business Objective

Enable smooth in-page navigation between vendor store listings and individual store pages via AJAX, loading content sections and re-initializing the media player without triggering full page reloads.

## Actors

- **Buyer/Visitor** — navigates between vendors or from listing to store page in the browser
- **System** — intercepts navigation events, fetches content, and updates DOM + player state

## UX Journey / User Actions

1. User is on a store listing page or a vendor store page
2. User clicks a vendor card → JS intercepts click (no browser navigation)
3. JS calls `GET /wp-json/tm-media-player/v1/store-content?vendor_id=N` (tmp-003 endpoint)
4. On success: injects returned playlist into media player; injects section HTML into DOM regions
5. Pushes new URL to browser history via `history.pushState`
6. Media player re-initializes with new vendor's playlist
7. On back navigation: JS listens to `popstate` → restores previous vendor content
8. On network error: falls back to `window.location.href = vendor_url` (full page reload)

## Inputs

| Field | Type | Source |
|---|---|---|
| vendor_id | int | Resolved from clicked vendor card DOM or URL |
| vendor_url | string | Vendor store page URL (for fallback reload) |

## Outputs

| Output | Description |
|---|---|
| Updated DOM | Store section content replaced in target regions |
| Player re-init | Media player JS re-initialized with new playlist |
| URL updated | Browser address bar updated via `pushState` |
| History state | Previous vendor content restorable via back button |

## State Transitions

```
player_state:     idle → loading → playing (new vendor content)
navigation_state: listing → store_page (AJAX-driven, no reload)
                  store_page → store_page (vendor-to-vendor, AJAX-driven)
                  store_page → store_page (popstate restores previous vendor)
```

## Conditional Logic

- If `vendor_id` changes from current: fetch new content; else skip fetch
- If fetch fails (network error or non-200): fall back to full page reload via `window.location`
- `dokan_is_store_page()` or `tm_is_showcase_page()` used server-side to confirm page context (compatibility adapter)
- Vendor ID resolved from DOM `data-vendor-id` attribute on card, or `get_query_var('author')` on server

## Side Effects

- Browser history state updated (`history.pushState`)
- Media player re-initialization triggers any mounted player event listeners
- Prior player buffer/playback stopped before new playlist loaded

## Current Dependencies

| Dependency | Usage |
|---|---|
| Dokan `dokan_is_store_page()` | Server-side page context detection for asset enqueue decisions |
| Dokan `dokan_is_store_listing()` | Store listing page detection |
| `tm_is_showcase_page()` | Platform showcase page detection |
| `get_query_var('author')` | Vendor ID resolution from URL query vars |
| tmp-003 REST endpoint | `GET /wp-json/tm-media-player/v1/store-content` |
| TM media player JS API | Re-initialization and playlist loading interface |

## Core Contract Definition

```
navigateToVendor(vendor_id: int, vendor_url: string) → void
  // 1. Fetch loadVendorContent(vendor_id)
  // 2. On success: update DOM sections, re-init player, pushState(vendor_url)
  // 3. On failure: window.location.href = vendor_url (full reload)

loadVendorContent(vendor_id: int) → Promise<StoreContentResponse>
  // Calls tmp-003 REST endpoint
  // Returns resolved StoreContentResponse or rejects on network/HTTP error

restoreVendorContent(event: PopStateEvent) → void
  // Restores previous vendor content from history.state on back navigation
```

## Compatibility Adapter Behavior

- Page detection via `dokan_is_store_page()` + `dokan_is_store_listing()` + `tm_is_showcase_page()`
- Vendor ID from `get_query_var('author')` on store pages (Dokan URL convention)
- Template sections from Dokan theme paths loaded server-side (via tmp-003 sections param)
- AJAX navigation scoped to Dokan store URL structure

## Default WP Adapter Behavior

- Page detection via `is_author()` or `is_post_type_archive('tm_vendor')` (no Dokan functions needed)
- Vendor ID from WP author query var or `tm_vendor` CPT post slug
- Template sections from WP block-rendered template parts (via tmp-003 default adapter)
- AJAX navigation scoped to WP-native vendor archive URL structure

## Parity Oracle

| Assertion | Pass Condition |
|---|---|
| Smooth navigation | Vendor content loads without full page reload on card click |
| Player re-init | Media player initializes with correct playlist for clicked vendor |
| History | Browser back button restores previous vendor content |
| Error fallback | Network error triggers full page reload to correct vendor URL |
| Vendor ID accuracy | Resolved vendor_id matches clicked vendor |
