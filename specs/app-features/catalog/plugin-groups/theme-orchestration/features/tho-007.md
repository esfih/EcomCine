# tho-007 — Cinematic Header CTA Actions Cluster

## Business Objective

Preserve the current right-edge cinematic header CTA design as a reusable feature contract even when the live header composition changes, so a later Phase 2 UI pass can restore or evolve the action rail without losing the design and implementation details already built.

## Actors

- **Guest** — sees the cinematic header action rail on supported header surfaces
- **Logged-in user** — currently shares the same header shell and may later regain the CTA rail alongside account access
- **System** — renders the right-edge action buttons and count badges inside the cinematic header shell

## UX Journey / User Actions

1. Cinematic header renders on store, person, showcase, and listing surfaces.
2. The right-edge container `.tm-header-actions` mounts a compact action rail.
3. Four action buttons render in order: menu toggle, favorites, shopping cart, notifications.
4. Each action uses shared icon-button styling and count badge positioning.
5. Cart count reflects live WooCommerce cart state when available.

## Current Render Contract

### Source file

- `tm-store-ui/includes/hooks.php`

### Current icon set

| Action | Icon source | Current behavior |
|---|---|---|
| Menu | `TM_Icons::svg( 'bars', 'tm-header-icon-svg' )` | Toggle button, `aria-expanded="false"` |
| Favorites | `TM_Icons::svg( 'heart', 'tm-header-icon-svg' )` | Placeholder badge count `0` |
| Cart | `TM_Icons::svg( 'cart-shopping', 'tm-header-icon-svg' )` | Badge count reflects `WC()->cart->get_cart_contents_count()` |
| Notifications | `TM_Icons::svg( 'bell', 'tm-header-icon-svg' )` | Placeholder badge count `0` |

### Current markup snapshot

```php
<div class="tm-header-actions" aria-label="Header actions">
	<button class="tm-header-icon tm-header-toggle" type="button" aria-label="Menu" aria-expanded="false">
		<?php echo $header_icon_menu; ?>
	</button>
	<button class="tm-header-icon" type="button" aria-label="Favorites">
		<?php echo $header_icon_favorites; ?>
		<span class="tm-header-count">0</span>
	</button>
	<button class="tm-header-icon" type="button" aria-label="Shopping cart">
		<?php echo $header_icon_cart; ?>
		<span class="tm-header-count"><?php echo (int) $cart_count; ?></span>
	</button>
	<button class="tm-header-icon" type="button" aria-label="Notifications">
		<?php echo $header_icon_notifications; ?>
		<span class="tm-header-count">0</span>
	</button>
</div>
```

## Styling Contract To Preserve

### Source file

- `tm-store-ui/assets/css/vendor-store.css`

### Visual characteristics

- Shared right-edge flex rail via `.tm-header-actions`
- Compact dark-glass icon buttons via `.tm-header-icon`
- Gold icon color and count badges via `.tm-header-count`
- Mobile/tablet rules currently suppress the CTA rail

## Conditional Logic

- Cart count depends on WooCommerce availability and cart hydration.
- The menu button is styled as a header action and may be bound by later JS/mobile menu behavior.
- Favorites and notifications are currently placeholder actions with static `0` badges.

## Phase 2 Development Note

This feature is intentionally preserved as a deferred Phase 2/UI-polish work item. The live header may temporarily repurpose `.tm-header-actions` for account access, but the existing cinematic CTA icon rail remains the documented baseline for future recovery or refinement.
