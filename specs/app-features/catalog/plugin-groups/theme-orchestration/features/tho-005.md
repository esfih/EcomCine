# tho-005 — Social Metrics, Vendor Completeness, and Map Modules

## Business Objective

Provide standalone computation and rendering modules for vendor social metrics (platform links and engagement counts), profile completeness scoring, and optional map display on vendor store pages — delivered as include-level services decoupled from the main template loop.

## Actors

- **System** — includes modules from `theme/includes/`; computes and renders outputs
- **Vendor** — source of social/location meta; sees their completeness score
- **Buyer** — sees social metrics and map on vendor store page

## UX Journey / User Actions

1. Store page template includes relevant modules from `theme/includes/` (e.g., `require_once get_stylesheet_directory() . '/includes/social-metrics.php'`)
2. **Social metrics module**: reads `tm_social_*` user meta → renders social platform icons and links block
3. **Completeness module**: reads profile meta fields → computes weighted score → renders progress bar or score indicator (visible to vendor or public per config)
4. **Map module**: reads vendor location meta (`tm_vendor_location`, `tm_vendor_lat`, `tm_vendor_lng`) → if non-empty and feature flag enabled → loads Mapbox or Google Maps embed
5. Asset for map (Mapbox JS / Google Maps JS) conditionally loaded only when map renders (tho-002 asset policy)

## Inputs

| Field | Type | Source |
|---|---|---|
| vendor_id | int | Template context |
| tm_social_linkedin | string | User meta |
| tm_social_twitter | string | User meta |
| tm_social_instagram | string | User meta |
| tm_social_youtube | string | User meta |
| Additional tm_social_* | string | User meta |
| tm_vendor_location | string | User meta (human-readable address) |
| tm_vendor_lat, tm_vendor_lng | float | User meta (coordinates for map) |
| Profile meta fields | varies | User meta (for completeness) |

## Outputs

| Output | Description |
|---|---|
| Social metrics HTML | Icon + link block per platform; absent if no social meta |
| Completeness HTML | Progress indicator with score (0–100); missing field list |
| Map embed HTML | Mapbox or Google Maps embed; absent if no location meta or feature disabled |

## State Transitions

None — read-only display modules.

## Conditional Logic

- **Social metrics**: rendered only if at least one `tm_social_*` meta is non-empty; empty platforms silently skipped
- **Completeness**: visible to vendor (always) or public (configurable via `tm_completeness_public` option); score computed as weighted sum of filled profile fields
- **Map module**: rendered only if `tm_vendor_lat` + `tm_vendor_lng` are non-empty AND `tm_maps_enabled` site option is `1`
- Map JS (Mapbox/Google) enqueued conditionally; tho-002 asset policy dequeues map scripts on non-map pages

## Side Effects

- Map module: Mapbox or Google Maps JS enqueued when map renders (managed via tho-002)
- Completeness score may be cached as `tm_completeness_score` user meta on compute (performance optimization)

## Current Dependencies

| Dependency | Usage |
|---|---|
| `theme/includes/social-metrics.php` | Social links render module |
| `theme/includes/completeness.php` | Score computation and render module |
| `theme/includes/map.php` | Map embed module (if separate file) |
| `get_user_meta` | Social and location meta reads |
| `tm_social_*` user meta convention | Social link platform data |
| `tm_vendor_lat`, `tm_vendor_lng` user meta | Map coordinate data |
| Mapbox GL JS or Google Maps JS | Map rendering library (conditional) |

## Core Contract Definition

```
computeSocialMetrics(vendor_id: int) → SocialMetrics
SocialMetrics {
  platforms: {
    id: string,          // e.g., 'linkedin', 'twitter'
    url: string,
    label: string
  }[]
}

computeCompleteness(vendor_id: int) → CompletenessResult
CompletenessResult {
  score: int               // 0–100
  sections: {
    name: string,
    filled: bool,
    weight: int
  }[]
  missing_fields: string[]
}

renderMapEmbed(vendor_id: int, options: MapOptions) → HTML string | ''
MapOptions {
  width: string,
  height: string,
  zoom: int
}
// Returns '' if no coordinates or feature disabled
```

## Compatibility Adapter Behavior

- `theme/includes/*.php` direct includes in store page template
- Data sourced from WP user meta with `tm_*` convention
- Module output is plain PHP rendered HTML
- Completeness module reads Dokan-conventional meta keys alongside `tm_*` keys for legacy compatibility

## Default WP Adapter Behavior

- Same PHP modules, but data sourced from vendor CPT (`tm_vendor`) post meta instead of user meta
- Map block: WP native map block or custom `tm_vendor_map` block via `register_block_type`
- Completeness computed from CPT meta fields (same algorithm; data layer abstracted behind `getVendorProfileMeta()` from tho-003 contract)
- Social metrics, completeness, map usable as standalone blocks or as classic include functions

## Parity Oracle

| Assertion | Pass Condition |
|---|---|
| Social metrics | Same platforms rendered for identical meta state |
| Social absent | Social block absent when all `tm_social_*` meta empty |
| Completeness score | Same score for equivalent profile fill state |
| Map shown | Map renders when both lat/lng set and feature enabled |
| Map absent | Map absent when no coordinates or feature disabled |
| Map asset scoping | No Mapbox/Google JS loaded on pages without active map module (tho-002 policy enforced) |
