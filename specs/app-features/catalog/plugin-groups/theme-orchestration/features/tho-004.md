# tho-004 — Product/Store Vendor Identity Presentation Hooks

## Business Objective

Inject vendor identity blocks (name, avatar, store link) into WooCommerce product cards and Dokan store listing cards to visually brand each listing with its vendor's identity.

## Actors

- **System** — hooks into product/listing loop templates to inject vendor identity HTML
- **Buyer** — sees vendor identity (name, avatar, store link) on product cards and listing cards

## UX Journey / User Actions

1. Store listing page or product archive loads
2. Theme hooks into WooCommerce product loop (`woocommerce_after_shop_loop_item_title`) or Dokan store listing loop hook
3. Vendor ID resolved from current loop: `get_post_field('post_author', $product_id)` or Dokan store loop context
4. `dokan_get_store_info($vendor_id)` → name, banner; `dokan_get_store_url($vendor_id)` → store URL
5. Avatar resolved: Dokan vendor banner → fallback `get_avatar_url($vendor_id, ['size' => 80])`
6. HTML block rendered below/above product card title: vendor name + avatar + store link
7. Click on vendor identity → navigates to vendor store page

## Inputs

| Field | Type | Source |
|---|---|---|
| product_id | int | WC product loop `$product->get_id()` |
| vendor_id | int | `get_post_field('post_author', $product_id)` or Dokan store loop |

## Outputs

| Output | Description |
|---|---|
| Vendor identity HTML | Name + avatar + store link inline HTML injected into product/listing card |

## State Transitions

None — display injection only.

## Conditional Logic

- Only inject in WC product loop or Dokan store listing loop contexts
- Skip if `$vendor_id = 0` (product has no author or author is not a vendor)
- Avatar fallback chain: Dokan vendor banner thumbnail → WP `get_avatar_url($vendor_id)`
- Store link: `dokan_get_store_url($vendor_id)` → if empty → `get_author_posts_url($vendor_id)`
- Identity block absent from non-listing pages (single product, blog, etc.)

## Side Effects

None — display injection only.

## Current Dependencies

| Dependency | Usage |
|---|---|
| WC hook `woocommerce_after_shop_loop_item_title` | Product card injection point |
| Dokan `dokan_get_store_info($vendor_id)` | Vendor name and banner/avatar data |
| Dokan `dokan_get_store_url($vendor_id)` | Vendor store page URL |
| `get_post_field('post_author', $product_id)` | Vendor ID from product |
| `get_avatar_url` | WP fallback avatar |

## Core Contract Definition

```
projectVendorIdentity(vendor_id: int) → VendorIdentity

VendorIdentity {
  name: string
  avatar_url: string    // banner thumbnail or WP avatar URL
  store_url: string     // Dokan store URL or WP author archive URL
}

renderVendorIdentityBlock(vendor_id: int, context: 'product_loop' | 'store_listing') → HTML string
  // Renders name + avatar + store link HTML
  // Returns '' if vendor_id is invalid
```

## Compatibility Adapter Behavior

- `dokan_get_store_info()` for vendor name and banner
- `dokan_get_store_url()` for store URL
- `woocommerce_after_shop_loop_item_title` hook for product card injection
- Avatar: Dokan banner thumbnail → WP `get_avatar_url` fallback

## Default WP Adapter Behavior

- Vendor CPT `tm_vendor` post meta replaces `dokan_get_store_info()` for name/avatar
- Author archive URL (`get_author_posts_url`) replaces `dokan_get_store_url()`
- Same `woocommerce_after_shop_loop_item_title` hook (or equivalent product loop block context)
- Block render callback for block-based product grid: uses `VendorIdentity` projection from same core contract

## Parity Oracle

| Assertion | Pass Condition |
|---|---|
| Identity data | Same name, avatar, store URL for equivalent vendor state |
| Identity block present | Vendor identity block appears on product cards and store listing cards |
| Identity block absent | Block absent on non-listing/non-product-loop page contexts |
| Avatar fallback | WP avatar rendered when Dokan banner not available |
| Store link valid | Store link navigates to correct vendor store page |
