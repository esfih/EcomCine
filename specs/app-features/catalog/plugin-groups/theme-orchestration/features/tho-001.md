# tho-001 — Store/Listing Template Override Orchestration

## Business Objective

Override Dokan store page and store listing page templates with custom cinematic theme equivalents, controlling page layout composition, section ordering, and shortcode-based content injection.

## Actors

- **Visitor/Buyer** — sees custom-rendered vendor store page or listing
- **System** — intercepts Dokan template hierarchy; returns custom template file paths

## UX Journey / User Actions

1. User visits a Dokan vendor store URL (e.g., `/store/vendor-slug/`) or store listing URL
2. Dokan template loading fires; system intercepts via `template_include` or Dokan template filters
3. System serves custom template file from `theme/dokan/` directory instead of Dokan plugin default
4. Custom template controls: page structure (header, media player region, attributes, product grid, sidebar, footer)
5. Platform page (`page-platform.php`) uses full-width store listing layout with `[dokan-stores]` equivalent
6. Talent showcase templates (`template-talent-showcase.php` / `template-talent-showcase-full.php`) used for showcase surface

## Inputs

| Field | Type | Source |
|---|---|---|
| Template name string | string | Dokan template filter arg |
| Current page context | page type | WP global `$post`, query vars |
| Vendor ID | int | Dokan query var / author query |

## Outputs

| Output | Description |
|---|---|
| Custom template file path | PHP template file in `theme/dokan/` or `theme/` replacing Dokan default |
| Rendered page HTML | Custom cinematic layout with correct section composition |

## State Transitions

None — template routing only; no state written.

## Conditional Logic

- Dokan template override: if file exists in `theme/dokan/{template_name}` → use it; else fall back to Dokan plugin default
- Full platform page: `page-platform.php` template selected → full-width store listing layout, not Dokan default page template
- Showcase templates used when `tm_is_showcase_page()` returns true
- Section ordering per template: `[media_player_region, profile_attributes, product_grid, store_footer]` — custom ordering differs from Dokan default

## Side Effects

- All Dokan/WC template hooks fire within custom template context (sections remain hookable)
- Performance: custom templates may load heavier assets (cinematic player)

## Current Dependencies

| Dependency | Usage |
|---|---|
| Dokan template filter (path-based override) | `theme/dokan/` shadows Dokan `templates/` |
| WP `template_include` filter | Custom template routing for platform/showcase pages |
| `dokan_is_store_page()` | Store page detection |
| `tm_is_showcase_page()` | Showcase page detection |
| `theme/dokan/` directory | Custom store page template files |
| `theme/template-talent-showcase.php` | Showcase surface template |
| `theme/template-talent-showcase-full.php` | Full-width showcase template |

## Core Contract Definition

```
getStorePageTemplate(context: PageContext) → template_file_path
  // context.is_showcase  → template-talent-showcase.php or template-talent-showcase-full.php
  // context.is_store     → theme/dokan/store.php (or equivalent)
  // context.is_platform  → page-platform.php
  // fallback             → Dokan/WP default

getListingPageTemplate(context: PageContext) → template_file_path
  // Full-width listing → page-platform.php
  // Standard listing   → theme/dokan/store-lists.php (or equivalent)

SectionOrder ['media_region', 'profile_attributes', 'product_grid', 'store_footer']
```

## Compatibility Adapter Behavior

- `theme/dokan/` directory shadows Dokan plugin `templates/` via Dokan's template override mechanism
- `template_include` filter for platform and showcase page routing
- Dokan page detection (`dokan_is_store_page`, `dokan_is_store_listing`) for context routing

## Default WP Adapter Behavior

- Block theme template files in `theme/templates/`:
  - `templates/single-vendor.html` replaces `theme/dokan/store.php`
  - `templates/archive-vendor.html` replaces store listing template
- Template parts in `theme/parts/` for individual sections (media region, attributes, product grid, footer)
- `theme.json` for global layout configuration (full-width toggle controlled via template attribute)
- No Dokan template filter; WP block theme handles template routing natively

## Parity Oracle

| Assertion | Pass Condition |
|---|---|
| Section composition | Store page renders: media region → attributes → product grid → footer, in this order |
| Full-width layout | Platform/showcase page renders without sidebar in full-width column |
| Custom template priority | Custom template overrides Dokan/WP default on store and listing pages |
| Hookability | Dokan/WC action hooks still fire within custom template (sections remain extensible) |
| Non-store pages | Custom store templates not applied on non-store pages (shop, product, blog) |
